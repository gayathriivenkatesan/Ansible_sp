# MFT_SMTP_NotListening [Windows] and SMTP_25 Not listening 
## Description

To Check whether the Gateway is having the connectivity with the ports [22,443,8080] just update the ticket and close it. If it is not having the connectivity we need to check the Harmony Service if Harmony Service is stopped we need to start an recheck the connection of the gateway , after rechecking also its failed we need to transfer the case to last assignement group ,after recheck its sucesffully having the connection we can close the case..

If port is 25 .It will connect  to target server and perform ping check .If connection is successful close the ticket else transfer the case 

## Playbook & Common Scripts

- Ansible Playbook - ***[MFT_SMTP_NotListening](roles/MFT_SMTP_NotListening/tasks/main.yml)***

Based on the port number it will trigger the respective playbook from main.yml

## Workflow MFT 

- Connect to the particular Gateway with respect to the centralised server we are getting from ticket. 
- then after connecting fetch the respective servers for the gateway.
- Validate the connection for the particular gateway with the port which we got in ticket
  `Example:- ports are 22,443,8080
- If the connection is sucessfull
  - Update WorkNotes as connection is sucesfull.
  - close the ticket.
- Else
  - If not having the connection
    - check the Harmony services 
    - If the Harmony services are stoped for respective srevers of the gateway start the services , if service are running we can leave  
    - Again recheck the connection, if its sucesfull
    - update workontes and close the ticket.
  - else if not able to connect after Services running
    - Update the  log in worknotes.
    - Transfer the case to last Assignment Group.

## Workflow SMTP

Script will execute when port is 25 .Connect to target server and check the connectivity using port 25 
-If 
  Connection to target server is successful it will close the ticket 
-Else 
  Transfer the case to SMTP team
